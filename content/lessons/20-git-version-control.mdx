## 📌 1. 核心痛點

AI 改 Code 時有一個常見問題：**越改越壞，連基本功能都冇埋**。

點解？因為 AI 係直接覆蓋原文件，你冇辦法返轉頭。

## 📌 2. 版本控制 = 遊戲存檔

就好似打機一樣——每過一段就 save 一次，死咗可以 load 返上一個 save。

Git 就係你嘅「遊戲存檔系統」。

## 📌 3. 叫 AI 幫你做版本管理

你唔需要學 Git 指令，直接同 AI 講：

> 幫我做版本管理，每次改重要嘢之前幫我 commit 一次

AI 會自動：
1. 初始化 Git 倉庫
2. 每次改之前自動存檔
3. 你可以隨時叫佢「返轉頭」

## 📌 4. Git 基礎指令速查

| 指令 | 功能 |
|---|---|
| `git init` | 初始化倉庫 |
| `git add .` | 加入所有更改 |
| `git commit -m "描述"` | 存檔 |
| `git log` | 睇記錄 |
| `git reset --hard HEAD~1` | 返到上一個版本 |

> 你唔需要記呢啲——叫 AI 幫你就得。
